# HIPAA Readiness Checklist for NorthStar MVP

This checklist is intentionally practical. It is not legal advice.

## Administrative safeguards

- [ ] Name a security officer.
- [ ] Complete a documented risk analysis.
- [ ] Create a risk management plan.
- [ ] Maintain a risk register.
- [ ] Train all workforce members before PHI access.
- [ ] Maintain a sanctions policy.
- [ ] Document workforce access approvals and removals.
- [ ] Review access at least quarterly.
- [ ] Create an incident response plan.
- [ ] Create a breach notification procedure.
- [ ] Create a contingency plan.
- [ ] Test backups and restores.
- [ ] Create vendor review procedures.
- [ ] Maintain BAAs with all covered-entity clients.
- [ ] Maintain subcontractor BAAs with all ePHI vendors.

## Physical safeguards

- [ ] Require full-disk encryption on company devices.
- [ ] Require screen locks.
- [ ] Require remote wipe for lost devices.
- [ ] Avoid paper PHI where possible.
- [ ] Secure any workspace where PHI is accessed.

## Technical safeguards

- [ ] Enforce MFA/passkeys.
- [ ] Use HTTPS only.
- [ ] Use secure cookies in production.
- [ ] Encrypt data at rest.
- [ ] Encrypt data in transit.
- [ ] Use managed key storage.
- [ ] Maintain immutable or tamper-evident audit logs.
- [ ] Limit internal admin access.
- [ ] Mask PHI in support tools.
- [ ] Avoid PHI in email/SMS/push notifications.
- [ ] Avoid PHI in analytics, logs, and session replay.
- [ ] Add rate limiting.
- [ ] Add vulnerability scanning.
- [ ] Add dependency scanning.
- [ ] Add penetration testing before launch.

## Product/legal safeguards

- [ ] Terms of Service.
- [ ] Privacy Notice.
- [ ] HIPAA Notice/workflow if applicable through covered-entity relationships.
- [ ] Family Access Authorization language reviewed by counsel.
- [ ] Data deletion/export workflow.
- [ ] Data retention schedule.
- [ ] Support workflow for user access, amendment, and access logs.
- [ ] Prohibition on selling patient data.
- [ ] Prohibition on ad retargeting using health data.
